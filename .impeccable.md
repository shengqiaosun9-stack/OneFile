# OneFile Design Context

## Target audience
- OPC (one-person company) founders and early-stage solo builders.
- Secondary: visitors who discover public project cards and may become creators.

## Use cases
- Quickly convert messy idea text into a structured project profile.
- Browse public projects to learn expression patterns before creating own project.
- Share project links externally and drive recipients into account creation + project creation.

## Brand personality and tone
- Calm, focused, trustworthy, founder-first.
- Professional but lightweight; readable and action-oriented.
- CTA language should be clear, direct, and growth-loop oriented.

## Product constraints
- Current stack: Next.js + shadcn/ui frontend, FastAPI backend.
- Home experience must feel like a public project space, not an admin console.
- Single primary CTA per major viewport area.
- Mobile-first responsiveness and motion accessibility (`prefers-reduced-motion`) are required.
